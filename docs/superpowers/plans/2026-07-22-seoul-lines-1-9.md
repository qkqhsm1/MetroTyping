# Seoul Lines 1–9 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Seoul Lines 1–9 playable, with correct branch/direction behavior, Line 9-only local/express play, and the approved full-width Seoul overview.

**Architecture:** Extend the existing `Line` record with one optional service-pattern list used only by Line 9. Reuse the current graph router, quick-route generator, random shuffle, setup flow, PDF-derived SVG overlay, and eight-station gameplay map; no new dependency or routing engine is introduced.

**Tech Stack:** React 19, TypeScript, Vite, Vitest/Testing Library, CSS, SVG, Node.js extraction script.

## Global Constraints

- Canonical repository is `https://github.com/qkqhsm1/MetroTyping`, branch `main`.
- New route data must be traceable to current official operator or public-sector sources immediately before freezing.
- Only Seoul Line 9 displays `일반 / 급행`; random and route modes both obey that selection.
- Line 6 remains unavailable until its directed Eungam-loop and UI tests pass.
- The overview raster never receives hover-time filter or transform work.
- Desktop overview has no internal scrollbar; narrow touch screens retain pan/zoom access to readable station names.
- Gameplay displays at most eight stations and typing never waits for animation.
- No new package or speculative general service engine.
- Each task follows RED → GREEN → focused verification → commit.

```ts
function serviceFor(lineId: string, serviceId?: string) {
  const line = getLine(lineId)
  if (!serviceId) return undefined
  const service = line.services?.find(item => item.id === serviceId)
  if (!service) throw new Error('Invalid service')
  return service
}

export function getStations(lineId: string, serviceId?: string) {
  const line = getLine(lineId)
  const service = serviceFor(lineId, serviceId)
  return service
    ? [...service.sequence]
    : [...new Set([...line.sequences.flat(), ...(line.oneWaySequences?.flat() ?? [])])]
}
```
