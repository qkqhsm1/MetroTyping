# 서울 4·6호선, 고해상도 지도, 메인 사운드 개선 설계

## 목표

MetroTyping을 10개 지원 노선으로 확장하고, 공식 노선도의 확대 품질·메인 카피·사운드 첫인상을 함께 개선한다.

## 지원 노선 추가

### 서울 4호선

- 실제 수도권 전철 전체 운행 구간인 `진접 ↔ 오이도`를 지원한다.
- 역 순서와 명칭은 현재 공식 서울교통공사·코레일 자료 및 공식 수도권 노선도와 대조한 뒤 고정한다.
- 빠른 여행은 `진접 ↔ 오이도` 카드 하나만 제공하며 양방향 버튼을 둔다.
- 모든 역의 사용자 지정 출발·도착 선택은 유지한다.

### 서울 6호선

- `응암순환 + 신내` 전체를 지원한다.
- 순환 구간은 실제 운행 방향인 `응암 → 역촌 → 불광 → 독바위 → 연신내 → 구산`만 허용한다.
- 본선 `응암 ↔ 신내`는 양방향 경로로 계산한다.
- 빠른 여행은 `응암 ↔ 신내` 양방향 카드와 `응암순환` 단방향 카드만 제공한다.
- 사용자 지정 경로도 응암순환의 단방향 제약을 지키며, 존재하지 않는 역방향 경로를 만들지 않는다.

## 경로 모델

기존 `Line.sequences`와 `getRoute()`를 유지하면서 선택적인 단방향 시퀀스 메타데이터를 추가한다. 경로 그래프는 일반 시퀀스에는 앞·뒤 링크를 만들고, 6호선 순환 시퀀스에는 다음 역 링크만 만든다. 역 선택·랜덤 출제·검증에서 일반 및 단방향 시퀀스의 역을 모두 하나의 노선 역 집합으로 취급한다.

빠른 여행 UI는 양방향 또는 단방향 경로 배열을 렌더링할 수 있게 일반화한다. 노선별 버튼 조건문을 UI에 넣지 않고 데이터가 허용 방향을 선언한다.

## 메인 지도 품질

### 렌더링

- 현재 `5102×5102` JPG의 DPI 메타데이터만 변경하지 않는다. 웹 선명도는 픽셀 수와 렌더링 방식으로 결정된다.
- 공식 PDF에서 직접 고배율 렌더링한 고해상도 이미지 타일 피라미드를 만든다.
- 전체 보기에서는 축소 레벨을, 확대·노선 선택 상태에서는 해당 영역의 고해상도 타일만 로드한다.
- 타일은 최신 브라우저가 지원하는 고효율 포맷을 우선 사용하고, 실제 변환 결과의 글자 선명도와 전송 크기를 비교해 WebP 또는 AVIF를 선택한다.
- PDF→SVG 전체 지도는 한글 글꼴 윤곽선이 축소 배율에서 불균일하게 보여 기본 배경으로 사용하지 않는다.

### 상호작용

- 공식 PDF에서 추출한 SVG 노선 경로는 선택·포커스 하이라이트에만 사용한다.
- 4호선과 6호선의 정확한 PDF 색상 레이어를 기존 추출 스크립트에 추가한다.
- 노선을 선택하면 해당 노선의 경계 상자로 짧고 부드럽게 확대한다.
- 지도 확대·이동은 타이핑 입력과 상태를 공유하지 않으며 게임 시작 이후 입력을 지연시키지 않는다.
- 모바일에서는 필요한 타일만 요청하고 페이지 자체의 가로 넘침을 만들지 않는다.

## 메인 카피

서울 화면:

- 작은 제목: `노선 선택`
- 큰 제목: `어느 노선에서 시작할까요?`
- 설명: `노선을 고르고 타이핑 여행을 시작해 보세요.`

도쿄 화면:

- 작은 제목: `路線選択`
- 큰 제목: `どの路線から始めますか？`
- 설명: `路線を選んで、タイピングの旅を始めましょう。`

기존 `SELECT YOUR NETWORK`와 `서울을 달리는 방법.` 문구는 제거한다.

## 사운드

### 진입 차임

- 사용자가 메인 지도에서 노선을 처음 선택하는 순간 한 번만 재생한다.
- 실제 서울교통공사 녹음 또는 기존 멜로디를 복제하지 않고 Web Audio로 독자적인 짧은 차임을 합성한다.
- 차임은 선택한 노선색과 어울리는 동일한 브랜드 음형을 사용하되 노선마다 알아차리기 어려운 무작위 차이를 만들지 않는다.
- 음소거 상태에서는 재생하지 않는다.
- 브라우저 자동재생 제한을 피하기 위해 사용자 클릭 이벤트 안에서 시작한다.

### 기존 효과음

- 현재 피크 게인 `0.09`를 정확히 1.5배인 `0.135`로 높인다.
- 빠른 타이핑으로 여러 음이 겹칠 때 출력이 찢어지지 않도록 Web Audio 출력 상한을 적용한다.
- 키·정답·오답·완료의 현재 주파수와 파형 구분은 유지한다.
- 음소거와 컨텍스트 정리 동작을 유지한다.

실제 공공기관 음원은 개별 공공누리 허용 표시나 권리자의 명시적 허락이 확인되지 않으면 사용하지 않는다. 서울시 저작권 정책상 공공누리 표시가 없는 자료는 담당 부서와 사전 협의가 필요하다.

## 실패 처리

- 단방향 경로가 존재하지 않으면 반대 방향을 추측해 만들지 않고 사용자에게 선택 불가 상태를 보여준다.
- 지도 타일 하나가 실패하면 저해상도 전체 지도를 유지해 노선 선택 기능은 보존한다.
- SVG 하이라이트 로딩 실패가 노선 카드 선택을 막지 않는다.
- AudioContext 생성·재개가 실패해도 게임과 노선 선택은 정상 진행한다.

## 검증

- 4호선 `진접 → 오이도` 전체 순서와 역방향을 테스트한다.
- 6호선 본선 양방향과 응암순환 단방향을 테스트하고 허용되지 않은 역방향을 거부한다.
- 4호선 빠른 여행 1개, 6호선 빠른 여행 2개와 각 허용 방향을 테스트한다.
- 신규 두 노선의 PDF 하이라이트 경로 수, 색상, 지도 정렬을 확인한다.
- 지도 타일과 SVG 좌표의 경계·확대 정렬을 수치로 검사한다.
- 360·768·1440 CSS 픽셀에서 전체 지도와 확대 상태를 캡처해 역명 선명도, 겹침, 페이지 넘침을 검사한다.
- 첫 노선 선택 차임, 음소거, `0.135` 게인, 출력 상한, 빠른 연속 입력을 테스트한다.
- `npm run check` 통과 후 GitHub Pages를 배포하고 새 번들·타일 MIME 및 HTTP 200을 확인한다.

## 공식 자료 기준

- 서울교통공사 역별/호선별 시간표 API 및 공식 수도권 노선도
- 코레일 4호선 운영 구간 자료: `오이도 ↔ 선바위`와 서울교통공사 공동 운행 자료
- 4호선·6호선 역명과 방향은 구현 직전에 최신 공식 자료로 다시 확인한다.
- 지도 원본은 프로젝트에 포함된 2025-09-29 서울 공식 PDF를 사용하고 공공누리 제1유형 출처표시를 유지한다.
